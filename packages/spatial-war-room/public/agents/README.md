# Agent Portraits · Visual Identity Log

Cute neko-doll chibi portraits for all 9 roster chairs in the Sovereign
Agentic Mesh. Generated by AG Nyx (Antigravity · Gemini 3 Pro) and
post-processed for asset hygiene + payload size.

## Pipeline

1. **Generate** at 512 × 512 with the per-agent prompts listed below.
2. **Strip metadata** — every image is re-rendered via a clean Pillow
   canvas to discard EXIF / `tEXt` / `iTXt` / source-prompt chunks and
   any embedded camera or tool path. Run on every regeneration.
3. **Color-quantize** with `pngquant --quality=70-92 --speed=1 --strip
   --skip-if-larger 96` (heroes) / `--skip-if-larger 64` (thumbnails).
   Pngquant respects perceptual color distance, so the chibi gradients
   survive far better than naive 128-color quantization in PIL.
4. **Budget**: heroes ≤ 92 000 bytes (≈ 90 KB), thumbnails ≤ 8 000
   bytes (≈ 8 KB). Both numbers are real ceilings — files that overshoot
   fail review. The check script below is the source of truth for what
   "overshoot" means; the prose ceilings above are just rounded
   restatements of the same numbers.

Asset budget verification:

```bash
for f in packages/spatial-war-room/public/agents/*.png; do
  size=$(stat -c%s "$f")
  case "$f" in
    *@128.png) max=8000 ;;
    *)         max=92000 ;;
  esac
  [ "$size" -le "$max" ] || echo "OVER BUDGET: $f ($size > $max)"
done
```

## Roster prompts

Each prompt below was the literal text fed to AG Nyx's image generator.
The neko / chibi style is deliberately consistent across agents — only
the **identity tells** vary (ear color, garment, accessory). This is
what makes the avatars instantly readable at the 36 × 36 size used in
the cockpit's roster cards.

A small `AgentIdentityBadge` component overlays a per-agent SVG glyph
on the bottom-left corner of each avatar in the UI; that glyph is the
same identity tell as the in-portrait accessory, so the differentiation
survives even when the painterly detail is too small to see.

## Regeneration recipe

To regenerate any single portrait, AG Nyx:

1. Uses the literal prompt above (no operator handle, no biographical
   reference — public-repo PII discipline).
2. Generates at 512 × 512 PNG with transparent background OR a solid
   `#1f1f1d` dark-grey background.
3. Downscales to 128 × 128 using Lanczos for the thumbnail.
4. Runs both through the pipeline above (strip → pngquant).
5. Commits with the path-anchored diff (`git add packages/spatial-war-room/public/agents/<id>.png`).

## Why we re-quantized in May 2026

The first-pass renders cleared `< 150 KB` per the brief's relaxed
budget, but the brief's *original* ceiling was `< 80 KB`. We tightened
to `pngquant --quality=70-92` (no perceptible quality loss on chibi
gradients), brought all 9 heroes well under 90 KB, and re-encoded the
thumbnails at the same time so they sit between 4 and 8 KB.
